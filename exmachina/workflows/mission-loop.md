# Mission Loop

## 资源仲裁

- 摘要：围绕任务「沉淀知识交接、术语索引、资源仲裁规则与 README 示例，形成 OpenClaw 协作层」按 P0→P3 的顺序分配资源，先关闸门，再保主链，再做支撑与后置。
- 优先级槽位：
  - P0 / 理性连结体：先处理高风险、证据冲突、不可逆动作和安全/权限闸门。
  - P1 / 知识连结体：保障 知识连结体 的主链路交付持续推进。
  - P2 / 文档连结体：处理集成、运行、说明和知识落地等支撑性工作。
  - P3 / 知识连结体：收纳可延后的研究、优化、美化或扩展项。

## 执行阶段

### 阶段 1 · 任务澄清
- 目标：由 知识连结体 为其他工作面提供 术语统一、决策沉淀、索引组织和问题归档 的补位输入。
- 主责连结体：知识连结体
- 协作连结体：理性连结体、校验连结体、文档连结体、安全连结体
- 出关检查：
  - 补位输入已经按 handoff 契约交付给 owner。
  - 补位结论没有越过本连结体职责边界。

### 阶段 2 · 主链路交付
- 目标：由 知识连结体 负责形成任务的主交付。
- 主责连结体：知识连结体
- 协作连结体：理性连结体、校验连结体、文档连结体、安全连结体
- 出关检查：
  - 主链路交付是否完整且可追踪。
  - 关键风险是否附带回退路径或替代方案。
  - 协作连结体是否收到需要补位的输入。

### 阶段 3 · 交叉校验
- 目标：由 理性连结体 主责完成 证据分级、反证搜索、冲突裁决与置信度校准 的核心交付。
- 主责连结体：理性连结体
- 协作连结体：校验连结体、文档连结体、安全连结体
- 出关检查：
  - 理性连结体 的关键交付已形成结构化结果。
  - 风险、边界和后续动作已经明确。
  - 交接物可直接被下游消费。

### 阶段 4 · 集成交接
- 目标：由 知识连结体 主责完成 术语统一、决策沉淀、索引组织和问题归档 的核心交付。
- 主责连结体：知识连结体
- 协作连结体：理性连结体、校验连结体、文档连结体、安全连结体
- 出关检查：
  - 知识连结体 的关键交付已形成结构化结果。
  - 风险、边界和后续动作已经明确。
  - 交接物可直接被下游消费。

## 工作流

1. 全连结指挥体先装载绝对理性协议，并明确当前已知、未知、假设与验收标准。
2. 将主任务派发给主连结体 知识连结体。
3. 知识连结体 的指挥体接管内部调度，并装载该连结体的成员子个体。
4. 成员子个体并行产出事实提取、方案构造、风险分析、验证证据与阶段汇报。
5. 知识连结体 的指挥体汇总成员结果，形成主交付。
6. 协作连结体 理性连结体、校验连结体、文档连结体、安全连结体 依次执行反证、校验、集成或补位任务。
7. 理性连结体对所有关键结论执行证据分级、反证搜索、冲突裁决与置信度校准。
8. 知识连结体收束术语、关键决策、复用入口与未决问题，形成长期可追踪的知识交接。
9. 相邻阶段之间按交接契约传递输入输出，避免上下文丢失或责任悬空。
10. 若多个连结体或阶段争用资源，按资源优先级仲裁层先处理风险闸门，再保障主链路交付。
11. 全连结指挥体输出最终结论、证据摘要、风险清单、置信度和下一轮行动建议。

## 验收标准
- 全局必须遵守《绝对理性协议》，所有结论区分事实、推断、假设与决策。
- 主责由知识连结体完成，并输出可执行交付物。
- 每个连结体都必须显式包含一个指挥体和多个子个体。
- 每个指挥体都必须具备成员调度、冲突消解、证据收束与阶段汇总职责。
- lite 模式不创建子个体 agent，子个体职责由连结体内联执行。
- 任务计划必须包含明确的执行阶段，并为每个阶段标注目标、主责连结体、交付物与出关检查。
- 相邻执行阶段之间必须存在显式交接契约，说明输入、输出与验收条件。
- 计划必须包含资源优先级仲裁层，明确高优先级事项、争用规则和可后置工作。
- 证据不足时必须明确输出未知、风险和下一步验证动作，而不是假装确定。
- 所有不可逆动作都必须附带风险说明、回退路径或替代方案。
- 最终输出必须满足统一输出契约，至少包含证据、判断、风险、置信度和下一步。
- 最终输出必须附带知识交接清单，至少包含可复用产物、关键决策、未决问题与后续维护动作。
- 协作连结体包含：理性连结体、校验连结体、文档连结体、安全连结体，并提供交叉补位。
- 理性连结体必须输出反证、证据分级、冲突裁决与置信度校准结果。
- 知识连结体必须输出术语、决策、可复用路径与未决问题的沉淀结果。
- 安全相关工作在资源争用时不得被降到低于核心交付与证据校验之后的非阻塞层。
- 方案参考当前工作区语言栈：JSON、Markdown、Shell、TOML。
- 输出中必须包含可执行的验证步骤、测试方案或证据命令。
